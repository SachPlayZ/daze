LANDING CLEANUP + TEAM FLAGS + FORMATION PITCH UI

CONTEXT
User asked 3 things after asking where Telegram integration lives (answered inline,
no code change -- bot is apps/bot/src/main.ts, long-polling, /link flow via
frontend/app/link/page.tsx + frontend/app/api/telegram/link/route.ts, no webhook/TWA SDK):

1. Landing page (frontend/app/page.tsx) embeds the historical Replay Builder directly
   (hardcoded fixture 18175981), duplicating what /fixtures and /replay/[fixtureId]
   already do properly. Remove it from landing; keep it only on its dedicated pages.

2. Team names (in fixtures list + replay builder) and each player's row have no country
   flag, even though every "team" in this World Cup fantasy app IS a national team --
   confirmed via tests/provider-fixtures/txline-devnet/scores-historical-18175981.json:
   Lineups[].preferredName (team name, e.g. "France") equals player.country. So one
   name -> flag lookup serves both team headers and player rows.
   DECISION: use the "flag-icons" npm package (real SVG/CSS flags by ISO 3166-1
   alpha-2 code), NOT emoji flags. Installed via pnpm (repo is a pnpm workspace --
   npm install breaks it, use pnpm add). Still need a country-name -> ISO2 lookup
   table since TxLINE only gives country names, not codes.

3. The player picker (ReplayBuilder.tsx) currently renders 4 stacked groups
   (GK/DEF/MID/FWD) with no visual formation. Replace with a two-pane layout: a CSS
   football pitch on the left showing circular slots per the selected formation
   (filled with team flag + last name once a player is picked), and the existing
   grouped player list on the right used to make picks.

3. FORMATION PITCH (replaces the 4-group-only view)
New file: frontend/app/components/FormationPitch.tsx
- Pure/derived, no internal state. Props: formation, playerIds, players, participants,
  onSlotClick (reuses existing togglePlayer to deselect).
- Local formationLayout mirrors packages/domain/src/index.ts formationCounts (GK
  always 1; DEF/MID/FWD per formation) -- duplicated client-side deliberately,
  consistent with existing pattern where ReplayBuilder.tsx already redefines
  Position/Formation locally instead of importing the domain package.
- Per position row (top->bottom: FWD, MID, DEF, GK), compute slots by filtering
  playerIds (existing selection order, togglePlayer appends) down to that position,
  slice to the row's count, pad remaining with null.
- Filled slot = circle with team flag centered inside + last name
  (preferredName.split(",")[0]) as a label above; click calls onSlotClick(id) to
  deselect. Empty slot = dashed circle showing position code, not clickable
  (selection stays driven from the list on the right).
- Pure CSS pitch background (green gradient stripes + halfway line + center circle),
  no image asset.

Integration in ReplayBuilder.tsx:
- Wrap .player-groups + new <FormationPitch /> in a new .builder-layout two-column
  flex/grid (pitch ~320-380px fixed left column, list flexes right; stacks to one
  column under 720px per existing mobile breakpoint pattern).
- Everything else (formation select, Quick Pick, captain/vice-captain, validate,
  entry flow) unchanged.
